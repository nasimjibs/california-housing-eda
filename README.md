# california-housing-eda

## Självutvärdering

### 1. Har något varit utmanande i kursen/kunskapskontrollerna?
Ja. Korrelationer tog ett tag att förstå — inte formeln, utan vad siffrorna faktiskt betyder i praktiken. Feature engineering var också nytt för mig. Att rådata i sig inte alltid berättar hela historien, och att man måste skapa nya kolumner för att hitta mönstren, var ett tankesätt jag fick öva på. Jag körde koden steg för steg och stannade upp när något inte stämde. EDA är mer iterativt än jag trodde — man vet inte vad man letar efter förrän man hittar det.

### 2. Vilket betyg anser du att du ska ha och varför?
VG. Jag har gått igenom hela flödet — saknade värden, feature engineering, flera olika visualiseringar med både matplotlib och seaborn, och dragit slutsatser som faktiskt går att motivera med data. Det är inte bara kod som körs utan fel, utan en analys där man kan följa resonemanget från rådata till insikt.

### 3. Något du vill lyfta fram till Terese?

Kombinationen av praktisk kodning och diskussionsfrågor om etik fungerade bra. Det är lätt att fastna i syntax — de här frågorna tvingade mig att tänka på vad tekniken faktiskt används till. Ser fram emot kurs 2.
